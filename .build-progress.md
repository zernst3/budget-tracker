**Why a Dioxus form, not a chorale editable Table (documented decision):** triage

multi-field, validated-on-submit shape fits plain `<select>`/`<input>` cleanly;
chorale's per-cell `EditorKind` (one committed value per cell, no cross-cell
conditional reveal) would fight it. The chorale `EditorKind::Select` IS used on
the ledger screen where the two editable fields are independent single-value
cells — the right tool there. The prompt explicitly allowed either; this is the
"your call, documented" choice.
